Le pouvoir de vote est établi à partir des REG possédés, quelque soit : 

  • la façon dont ils sont utilisés : sur son Wallet,  déposé sur une Pool de liquidité ou déposé sur le Vault d'incentive,
  • la blockchain utilisée (Gnosis, Polygon ou Ethereum).

Avant un vote, à une date non prévisible (afin qu'il n'y ait pas de manipulation), une photo (snapshot) est executée pour collecter la situation de tous les wallets possédant des REG.
A partir de cette collecte, un programme (offChain) calcule le Pouvoir de vote de chacun. Il s'agit d'une pondération, suivant le mode d'utilisation des REG (et des contre-parties dans les pools de liquidité), qui est décidée par la DAO.
A l'issue du calcul, le nombre de token powerVoting est actualisé.

La quantité de token powerVoting de chacun reste inchangée, jusqu'au prochain snapshot (même si la quantité de REG a bougée entre temps). Ce token est non transférable. 
La quantité de powerVoting d'un wallet est visible sur l'application Tally, ou sur l'explorateur 

Programme de calcul

Le programme, réalisé par RealT, pour le snapshot et le calcul  est disponible sur le Github de la Communauté.
Son fonctionnement est détaillé au chapitre 4.4 de cette page

A l'occasion de chaque snapshot, deux types de fichier sont générés (au format csv et json) : 

  • balancesREG : Liste tous les wallets qui possèdent des REG, avec la quantité pour chaque mode de d'utilisation,
  • powerVotingREG_balances : Liste des wallets qui possèdent des PowerVoting avec la quantité résultant du calcul par l'algorithme.
    Sont exclus de cette seconde liste (car ils ne peuvent pas voter) les adresses : des Pool de liquidité, du Vault d'incentive, des Safes qui gèrent les budgets de la DAO, ainsi que l'allocation à Mt Pelerin.

Les fichiers sont communiqués à la Communauté pour validation avant application sur le Telegram de la DAO : Exemple

Un Comité dédié à l'évolution de l'algorithme : [RIP00007]. Les échanges du Comité sont visibles (en lecture seule) sur ce Telegram

Algorithme de pondération

Version 1 (Octobre 2024)

Les premiers snapshots ont utilisé, la pondération suivante : 

  • 1 REG dans un Wallet ou le Vault d'incentive vaut 1 PowerVoting,
  • 1 REG dans un pool de liquidité vaut 1,2 PowerVoting,
  • Les stablecoins apportés en contre partie dans une Pool de liquidité valent 0,5 PowerVoting de l'équivalent REG.

Ce premier algorithme a été établi par RealT, les suivants seront votés par la DAO.
 

Version 2 (Mars 2025)


Les modifications sont les suivantes : 

  • nouvelle pondération, suivant l'usage des REG (cf image )
  • les montants de REG sont la moyenne des REG détenus depuis le précédent snapshot,
  • les Pouvoirs de Vote <1 seront considérés comme nuls.
     

Voté par la  RIP00018. ces modifications ont été appliquées à partir de la RIP00022.
 

Version 3 (Février 2026)


L'objectif de cette version est de modifier la pondération des pools de REG, afin qu'elle soit plus en cohérence avec les intérêts de la DAO.
La pondération sera maintenant d'autant plus forte, que la liquidité sera proche du prix (et donc utile pour limiter les glissements lors des échanges).

Cette version a été l'occasion du transfert du code vers le Github communautaire (et donc la gestion associée).
Une documentation spécifique à ce nouveau mode de calcul est disponible sur Github.

Voté par la RIP00037 ,les modifications ont été appliquées à partir de la RIP00039.
