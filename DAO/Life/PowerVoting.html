Voting power is determined by the number of REGs held, regardless of:

  • how they are used: in a wallet, deposited in a liquidity pool, or deposited in the incentive vault;
  • the blockchain used (Gnosis, Polygon, or Ethereum).

Before a vote, on an unpredictable date (to prevent manipulation), a snapshot is taken to capture the status of all wallets holding REGs. 
From this data, an off-chain program calculates each holder's voting power. 
This power is weighted according to the REG usage method (and the corresponding liquidity pool counterparties), as determined by the DAO.
After the calculation, the number of powerVoting tokens is updated.

Each holder's powerVoting token quantity remains unchanged until the next snapshot (even if their REG quantity has changed in the meantime). This token is non-transferable.The quantity of Voting Power of a wallet is visible on the application Tally, or  explorer.

Calculation program

The program, created by RealT, for the snapshot and calculation is available on Community Github
Its operation is detailed in chapter 4.4 of this page

For each snapshot, RealT generates two types of files (in csv and json format):

  • balancesREG: Lists all wallets that have REG, with the quantity for each mode of use,
  • powerVotingREG_balances: List of wallets that have PowerVoting with the quantity resulting from the calculation by the algorithm. The following addresses are excluded from this second list (because they cannot vote): Liquidity Pools, Incentive Vaults, Safes that manage the DAO budgets, and the allocation to Mt Pelerin.

The files are communicated to the Community for validation before application on the DAO Telegram: Example

Weighting algorithm

Version 1 (October 2024)

The first snapshots used the following weighting:

  • 1 REG in a Wallet or the Incentive Vault is worth 1 PowerVoting
  • 1 REG in a liquidity pool is worth 1.2 PowerVoting,
  • The stablecoins brought in return in the Liquidity Pool are worth 0.5 PowerVoting of the REG equivalent

Version 2 (March 2025)

 

The changes are as follows : 

  • New weighting, based on REG usage (see image).
  • REG amounts are the average of REG held since the previous snapshot.
  • Voting Powers <1 will be considered null.

Approved by RIP00018, the changes were implemented starting with RIP00022.
 

Version 3 (February 2026)

 

The goal of this version is to modify the weighting of REG pools to better align with the DAO's interests. The weighting will now be higher the closer the liquidity is to the price (and therefore useful for limiting slippage during trades). 

This version also marked the transfer of the code to the community GitHub repository (and thus the associated management). 
Specific documentation for this new calculation method is available on GitHub. 

Approved by RIP00037, the changes were implemented starting with RIP00039
